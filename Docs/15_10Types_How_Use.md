## 📝 초기 스타트업 마케팅 전략: 프롬프트 유형별 분석 보고서

## 상황 요약 (Context)

> **"최근 앱 출시 후 초기 사용자는 모았으나, 경쟁사 앱의 공격적인 마케팅으로 인해 사용자 증가세가 둔화되었습니다. 다음 분기 마케팅 전략을 새롭게 수립해야 하는 상황입니다."**
>
> **전문가 진단:** 이는 초기 제품 시장 적합성(Product-Market Fit)을 확보했으나, **'외부 위협(T)'**에 대한 **'내부 자원(W)'**이 부족하여 성장 모멘텀을 잃을 위험(Risk)에 직면한 전형적인 스타트업의 성장통입니다. 전략적 대응을 위해 AI에게 명확하고 구조화된 분석을 요청해야 합니다.

## 1. 지시형 (Instructional) 프롬프트 분석

### 상세 설명 및 예시 분석

| 구분 | 내용 |
| :--- | :--- |
| **상세 설명** | AI에게 명확하고 직접적인 지시를 내려 특정 작업을 빠르게 수행하도록 요구하는 가장 기본적인 유형입니다. **'~해 줘', '~작성해 줘'** 등 명령형 동사를 사용합니다. |
| **예시 (P1)** | **"경쟁사 공격으로 성장 둔화를 겪는 초기 스타트업을 위한 다음 분기 마케팅 전략 3가지를 핵심만 요약해서 작성해 줘."** |
| **분석** | 상황 배경은 제시했으나, **어떤 정보를 기반**으로, **어떤 형식**으로 전략을 도출해야 하는지에 대한 **지침이 부재**합니다. AI의 일반적인 지식(General Knowledge)에 의존하게 됩니다. |

### SWOT 분석을 통한 해결 및 전략 도출

**SWOT 분석 관점 (기업 리스크 전문가 역할):** 지시형 프롬프트만으로는 깊이 있는 SWOT 분석이 불가능하므로, 전문가의 가정이 추가된 일반적인 전략이 도출됩니다.

| SWOT 요소 | 강점(S), 약점(W), 기회(O), 위협(T) |
| :--- | :--- |
| **S/W (내부)** | S: 초기 사용자 만족도 / W: 마케팅 예산 및 인력 부족 |
| **O/T (외부)** | O: 틈새 시장 잔존 / T: 경쟁사의 공격적 마케팅 |

### 유형별 결과 및 미비점

| 구분 | 결과 (예상 AI 응답) | 미비점 (Deficiencies) |
| :--- | :--- | :--- |
| **결과** | **1. 핵심 사용자 유지(Retention):** 기존 사용자에게 특별 혜택 제공. **2. 저비용 바이럴 마케팅:** 콘텐츠 마케팅 강화. **3. 틈새 시장 재정의:** 경쟁사가 커버하지 못하는 니치 타겟 공략. | 1. **깊이 부족:** 상황만 반영했을 뿐, 스타트업의 **구체적인 강점/약점을 반영하지 못합니다.** 2. **실행 가능성 모호:** '저비용 바이럴' 같은 **일반적인 답변**이 많아 실무 적용을 위한 구체적인 액션 플랜으로 이어지기 어렵습니다. |

## 4. 제약 조건 (Constraint-Based) 프롬프트 분석

### 상세 설명 및 예시 분석

| 구분 | 내용 |
| :--- | :--- |
| **상세 설명** | AI의 응답에 **특정 형식, 길이, 포함/제외 키워드** 등의 제약(Constraint)을 명시하여 원하는 규격에 맞는 출력을 강제하는 유형입니다. 복잡한 비즈니스 문서화에 유용합니다. |
| **예시 (P4)** | **"당신은 리스크 컨설팅 전문가입니다. 초기 스타트업의 '강점(S)', '경쟁사 위협(T)', '틈새 기회(O)', '예산 약점(W)'을 대입하여 SO, ST, WO, WT 전략 4가지를 도출하고, **반드시 Markdown 표 형식**으로 작성해 주세요. **각 전략의 설명은 50자 이내**로 제한합니다."** |
| **분석** | **역할 부여(전문가)**와 **출력 형식(표, 50자 제한)** 제약을 동시에 사용함으로써, AI에게 'SWOT Mix'라는 구조화된 사고를 강제하고, 답변의 품질과 실용성을 높입니다. |

### SWOT 분석을 통한 해결 및 전략 도출

**SWOT Mix 전략 도출 (AI가 수행하도록 유도):**

| 전략 유형 | 설명 (50자 이내) | 리스크 대응 |
| :--- | :--- | :--- |
| **SO (강점-기회)** | 초기 사용자 강점 기반 틈새 시장 공략을 위한 '커뮤니티 중심 성장' 전략 실행. | 성장 기회 최대화 |
| **ST (강점-위협)** | 경쟁사가 모방하기 어려운 초기 제품 만족도를 극대화해 위협을 회피 (방어). | 핵심 강점으로 위협 방어 |
| **WO (약점-기회)** | 예산 약점을 틈새 시장의 높은 효율성(ROI)으로 극복하는 '타겟 집중형 광고' 실험. | 약점 극복 기회 활용 |
| **WT (약점-위협)** | 예산 부족과 경쟁 위협 하에 **최소한의 자원으로 핵심 기능만 유지**하며 생존 모드 전환 (축소/철수 준비). | 리스크 최소화 |

### 유형별 결과 및 미비점

| 구분 | 결과 (구조화된 4가지 전략) | 미비점 (Deficiencies) |
| :--- | :--- | :--- |
| **결과** | 전략 수립의 **구조와 방향성(4가지 관점)**을 명확하게 제시하여, 리스크 전문가의 의사 결정 흐름과 일치하는 결과를 얻었습니다. 답변의 **정합성**이 높습니다. | 1. **입력 데이터 의존도:** S/W/O/T 요소 자체를 **사용자가 명확히 정의**해 주지 않으면 AI가 임의로 판단하여 결과가 흔들릴 수 있습니다. 2. **지나친 간결성:** 50자 제약 때문에 전략의 **구체적인 실행 방법**을 추가로 요청해야 합니다. |

## 9. 인스턴스 기반 (Instance-Based) 프롬프트 분석

### 상세 설명 및 예시 분석

| 구분 | 내용 |
| :--- | :--- |
| **상세 설명** | AI에게 특정 대상(인물, 제품, 경쟁사 등)의 **구체적인 데이터 또는 예시(Instance)**를 제공하고, 이를 중심으로 답변을 도출하도록 유도하여 **구체성과 맥락(Context)**을 확보하는 유형입니다. |
| **예시 (P9)** | **"우리 앱 'A'의 핵심 강점(Instance)은 'Z세대 사용자의 높은 커뮤니티 활동성'입니다. 이 강점을 활용하여, **경쟁사 앱 'Z'(Instance)의 공격적 마케팅을 무력화시킬 수 있는** 차별화된 다음 분기 콘텐츠 마케팅 캠페인을 3가지 제안해 줘."** |
| **분석** | 상황을 일반적인 '스타트업'이 아닌 **'Z세대 커뮤니티가 강점인 A 앱'의 리스크**로 구체화했습니다. AI는 이 **핵심 인스턴스**를 해결책의 중심축으로 활용하게 됩니다. |

### SWOT 분석을 통한 해결 및 전략 도출

**Instance를 활용한 ST 전략 도출 (강점 기반 위협 회피):**

1.  **S (강점):** Z세대 커뮤니티 활동성 (Instance)
2.  **T (위협):** 경쟁사 앱 Z의 공격적 마케팅
3.  **ST 전략:** S를 활용해 T를 무력화할 **'Z세대 대상 참여형 콘텐츠'**를 중점 개발.

### 유형별 결과 및 미비점

| 구분 | 결과 (예상 AI 응답) | 미비점 (Deficiencies) |
| :--- | :--- | :--- |
| **결과** | **1. Z세대 챌린지 릴레이:** 사용자가 참여하는 숏폼 챌린지 기획. **2. 핵심 유저 앰버서더 프로그램:** 커뮤니티 리더에게 독점 보상 제공. **3. 경쟁사 대비 '힙함' 강조:** 경쟁사 앱 Z의 딱딱함을 역이용하는 인스턴스 기반 차별화 콘텐츠. | 1. **확장성 제한:** 제시된 인스턴스(Z세대)에 **과도하게 집중**하여, 혹시 놓치고 있는 **다른 잠재적 시장(예: 3040세대)**에 대한 전략적 사고가 부족해질 수 있습니다. 2. **인스턴스 품질 의존:** 입력된 인스턴스(강점 데이터) 자체가 부정확할 경우, 도출되는 전략 전체가 실패할 위험이 높습니다. |
